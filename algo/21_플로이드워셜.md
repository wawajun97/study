모든 쌍 최단 경로 : 각 정점 사이의 최단 경로(음의 가중치도 존재)  
ex) 한 도시에서 다른 도시로 가장 빨리 갈 수 있는 경로를 찾는 문제 -> 가중치 포함, 방향성 그래프에서 최단경로 찾기  
1. 가중치가 있다 -> BFS
2. 양의 가중치만 있다 -> 다익스트라
3. 음의 가중치도 있다 -> 벨만포드, 플로이드워셜

플로이드워셜 적용  
- 무향 : 양의 가중치만 가능
- 유향 : 양의 가중치, 음의 가중치 가능(음의 싸이클이 없는 경우)

플로이드 알고리즘 시간 복잡도 : O(N^3)으로 다익스트라의 시간복잡도와 동일  

DP 접근 방법
1. 부분 문제를 찾는다 -> 그래프의 정점의 수가 적을 때를 생각
2. 정점 i에서 정점 j까지의 최단 경로를 찾는다.(중간에 몇 개의 노드를 거치는지는 상관없고 가중치가 최소가 되는 경로 구하기)
   - 반드시 모든 정점을 경유할 필요는 없다.
3. 정점 1로부터 시작하여 정점 1,2 그 다음엔 정점 1,2,3으로 하나씩 추가하여 마지막에는 정점 1~n까지의 모든 정점을 경유 가능한 정점들로 고려하면서 모든 쌍의 최단 경로의 거리 계산
4. D[i][j] = 경유 가능한 정점들로 고려하여 정점 i로부터 j까지의 모든 경로 중에서 가장 짧은 경로의 거리
    - D[i][j](1) = Math.min(D[i][j](0), D[i][1](0) + D[1][j](0)) -> (1)은 1을 경유지로 선택했을 때를 의미. 즉, 바로 가는 것과 1을 경유해서 가는 것을 비교하여 최소값 저장
    - 코드로 구현할 때는 그냥 2차원 배열로 구현 -> 어디를 거쳐서 가는게 최소인지 알 필요 없어서 최소값만 저장하면 되기 때문
    - 가는 경로가 없다면 100_000_000(최대값 -> Integer.MAX_VALUE로 하면 범위 넘어가서 부호 반전될 수 있음) -> 그래야 min에서 걸러짐  
=> i에서 j로 가는 경로를 구하는데 중간에 k를 거쳤을 때를 계산해야 하므로 3중 for문이 돈다.
=> 모든 D[i][j]가 최적이어야 D[i][k] + D[k][j] 계산이 되므로 k를 가장 바깥으로 하여 최적을 업데이트 해준다.

```java
//D[][] 배열 초기 : i == j => 0 / 두 노드가 연결이 안 되어있으면 100_000_000

for (int k = 0; k < N; k++) { //경유지
    for (int i = 0; i < N; i++) { //출발지
        for (int j = 0; j < N; j++) { //도착지
            D[i][j] = Math.min(D[i][j], D[i][k] + D[k][j]);
        }
    }
}
```
