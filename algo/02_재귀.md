# 재귀

재귀는 함수 내부에서 직접 또는 간접적으로 자기 자신을 다시 호출하는 방식이다.

- 기본 부분(base case): 더 이상 재귀 호출이 발생하지 않는 부분으로, 가장 작은 문제를 처리하는 구간
- 유도 부분(recursive case): 더 작은 문제로 나누어 재귀 호출이 이어지는 부분

## 재귀를 설계할 때 확인할 점

1. 함수가 무엇을 하는지 정의를 명확히 한다.
2. 자신의 작업과 나머지 작업을 분리해서 설계한다.
   - 하나의 함수에서 모든 일을 처리하려고 하지 않는다.
   - 부분 문제를 먼저 식별한다.
3. 함수의 동작을 결정하는 핵심 값이 무엇인지 찾는다.
   - 즉, 어떤 매개변수가 재귀의 흐름을 바꾸는지 확인한다.
4. 재귀의 기본 부분을 반드시 정한다.
   - 종료 조건이 없으면 재귀는 끝나지 않는다.

```java
public static void main(String[] args) {
		test(4);
	}

	private static void test(int size) {
		if(size == 0) return; //base case

		test(size-1);

		for(int i = 0; i <size;i++) {
			System.out.print("* ");
		}
		System.out.println();
	}
```

## 순수 함수(Pure Function)

- 같은 값을 인자로 넘기면 항상 같은 결과를 반환하는 함수이다.
- 이미 계산한 결과를 저장해 두었다가 다시 사용하는 기법을 메모이제이션이라고 한다.
