힙 : 완전 이진 트리에 있는 노드 중에서 키 값이 가장 큰 노드나 키 값이 가장 작은 노드를 찾기 위해서 만든 자료구조
- 최대힙 : 부모 노드의 키 값 >= 자식 노드의 키 값 -> 루트가 가장 큼
- 최소힙 : 부모 노드의 키 값 <= 자식 노드의 키 값 -> 루트가 가장 작음

최대힙에서 노드 추가 -> O(logN)
1. 마지막 노드 다음에 추가한다
2. 부모가 추가한 노드보다 작으면 부모와 변경한다
3. 부모가 더 클 때까지 2번을 반복한다

최대힙에서 노드 삭제(루트 노드만 삭제 가능 - O(logN))
1. 마지막 원소를 루트로 올린다
2. 자식 중 더 큰 값과 삽입한 원소와 바꾼다
3. 자식이 없거나 자식들이 모두 삽입한 원소보다 작을 때까지 2번을 반복한다

우선순위 큐 : 우선순위를 가진 항목들을 저장하는 큐
- 우선순위가 높은 순서대로 먼저 나감
- 힙으로 구현되어 있음
- 완전 이진 트리로 되어있어 인덱스로 접근 가능하다(왼쪽 자식 인덱스 = i*2 , 오른쪽 자식 인덱스 = i*2 + 1)
