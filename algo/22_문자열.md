문자열 X 안에 문자열 Y가 포함되어 있는지 확인하는 문제  
if X 길이가 100000 Y 길이가 100이라면? -> 완탐 불가능  

## KMP 알고리즘  
- 불필요한 탐색, 비교를 줄이는 방법
- 부분일치 테이블 생성 -> 길이별로 접두사와 접미사가 겹치는 최대 길이를 구해놓고 접두사 길이만큼은 체크하지 않는다.
- 실패했을 때 다음 탐색은 어디?

부분일치 테이블 만들기
```java
int[] table = new int[n];
		
int j=0;
for(int i=1; i<n; i++) {
  while(j>0 && pattern.charAt(i) != pattern.charAt(j)) {
    j = table[j-1]; //여기도 부분일치 테이블을 보고 앞에 맞은 만큼은 비교하지 않음
  }
    
  if(pattern.charAt(i) == pattern.charAt(j)) {
    table[i] = ++j;  
  }
}
```
방법
  - 본문(t배열) 인덱스(i)와 패턴(p배열) 인덱스(j)를 각각 0에서 부터 시작-
  - t[i] == p[j] -> i++,j++; 둘이 같지 않을 때까지 진행
  - 만약 같지 않다면 부분일치 테이블을 보고 접두사 길이만큼 j-k, i는 그대로 -> 패턴에서 틀리면 i++,j++인가?
  - 2번 반복하다가 i == N이면 문자열 없음, j == M이면 문자열 있음  


## 라빈 카프 알고리즘
- 문자열의 해시값을 비교
- 패턴의 해시 값과 본문 안에 있는 하위 문자열의 해시 값만을 비교  

방법
  - 본문에 i부터 i+패턴 길이 만큼의 해시값을 모두 계산한다. -> 해시 함수는 아무거나 사용(어차피 본문과 패턴에 동일한 해시함수만 사용하면 됨)
  - 패턴의 해시값을 계산한다.
  - 해시값이 같으면 패턴이 본문에 있다.(우연의 일치로 문자열이 다르지만 해시값은 같을 수 있지만 거의 없다)
