# 트리

트리는 비선형 자료구조이면서 계층형 자료구조이다.

- 원소들 사이에 1:N 관계가 존재한다.
- 상위 원소에서 하위 원소로 내려가며 확장되는 구조를 가진다.

## 기본 용어

- 노드: 트리를 이루는 각각의 원소
- 루트 노드: 최상위 노드
- 간선: 노드와 노드를 연결하는 선
- 형제 노드: 같은 부모를 가진 자식 노드들
- 조상 노드: 특정 노드에서 루트까지 올라가는 경로에 있는 모든 노드
- 서브트리: 부모와의 연결을 끊었을 때 만들어지는 하위 트리
- 자손 노드: 어떤 노드의 서브트리에 속한 하위 노드들
- 단말 노드: 차수가 0인 노드, 즉 자식이 없는 노드
- 내부 노드: 루트와 단말 노드를 제외한 나머지 노드

## 차수

- 노드의 차수: 해당 노드가 가진 자식 노드의 수
- 트리의 차수: 트리 안의 노드들 중 가장 큰 차수

## 높이

- 노드의 높이: 루트에서 해당 노드까지 오는 간선 수
- 트리의 높이: 트리 안의 노드들 중 가장 큰 높이

## 이진 트리

이진 트리는 각 노드가 최대 두 개의 자식을 가지는 트리이다.

- 왼쪽 자식과 오른쪽 자식으로 구분한다.
- 높이가 `h`인 이진 트리의 최소 노드 수는 `h + 1`이다.
- 높이가 `h`인 이진 트리의 최대 노드 수는 `2^(h+1) - 1`이다.

### 이진 트리 종류

- 정 이진 트리: 모든 노드의 차수가 0 또는 2인 트리
- 포화 이진 트리: 모든 레벨이 가득 찬 트리
- 완전 이진 트리: 마지막 레벨을 제외한 모든 레벨이 가득 차 있고, 마지막 레벨은 왼쪽부터 빈칸 없이 채워진 트리
- 편향 이진 트리: 한쪽 방향으로만 자식이 이어지는 트리

## 이진 트리 번호

- 루트부터 시작해서 왼쪽에서 오른쪽 순서로 번호를 매긴다.
- 레벨 단위로 번호를 붙인다고 생각하면 된다.
- 노드 번호가 `i`이면 부모 노드 번호는 `i / 2`이다. 이때 루트 번호는 1이어야 한다.
- 노드 번호가 `i`이면 왼쪽 자식 번호는 `i * 2`이다.
- 노드 번호가 `i`이면 오른쪽 자식 번호는 `i * 2 + 1`이다.
- 레벨 `n`의 시작 번호는 `2^n`이다.
- 따라서 노드 번호를 배열 인덱스로 사용할 수 있다.

## 이진 트리 순회

1. 전위 순회(preorder): 부모 -> 왼쪽 -> 오른쪽
2. 중위 순회(inorder): 왼쪽 -> 부모 -> 오른쪽
3. 후위 순회(postorder): 왼쪽 -> 오른쪽 -> 부모

```java
public void dfsByPreOder(int current) { // 부모 -> 좌 -> 우 : 전위순회
		System.out.println(nodes[current]);

		if (current * 2 <= lastIndex) dfsByPreOder(current*2);
		if (current * 2 + 1 <= lastIndex) dfsByPreOder(current*2 + 1);
	}

	public void dfsByInOder(int current) { // 좌  -> 부모 -> 우 : 중위순회
		if (current * 2 <= lastIndex) dfsByInOder(current*2);
		System.out.println(nodes[current]);
		if (current * 2 + 1 <= lastIndex) dfsByInOder(current*2 + 1);
	}

	public void dfsByPostOder(int current) { // 좌 -> 우 -> 부모 : 후위순회
		if (current * 2 <= lastIndex) dfsByPostOder(current*2);
		if (current * 2 + 1 <= lastIndex) dfsByPostOder(current*2 + 1);

		System.out.println(nodes[current]);
	}
```
