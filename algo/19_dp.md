1. 재귀 호출 -> 하향식 접근 설계
2. 메모이제이션 -> 이전 계산을 기억하고 재사용 -> 중복연산 발생한다    

ex) 피보나치 수열  
1. 재귀 => 중복 호출이 많이 일어남    
```java
public static int fibo(int n) {
  if(n < 2) return 1;

  return fibo(n-1) + fibo(n-2);
}
```  

2. 메모이제이션 => 추가적인 메모리 공간이 필요하다(memo 배열), 재귀 호출로 인한 스택 오버플로우 가능성 존재
```java
public static int fibo(int n) {
	if(memo[n] != -1) return memo[n];
	
	memo[n] = fibo(n-1) + fibo(n-2);
	
	return memo[n];
}
```

DP : 그리디 알고리즘과 같이 최적화 문제를 해결하는 알고리즘  
=> 작은 부분 문제들의 해들을 구하고 이들을 이용하여 보다 큰 크기의 부분 문제들을 해결하여 최종적으로 원래 주어진 문제를 해결  

dp 조건
1. 중복 부분문제 구조 => 일반적으로 점화식을 사용 -> 중복 계산 피함
2. 최적 부분문제 구조 or 경우의 수 => 최적화의 원칙을 만족해야 함 => 어떤 문제에 대한 해가 최적일 때 그 해를 구성하는 작은 문제들의 해 역시 최적이어야 한다.  
=> 더 작은 부분문제를 이용한다는 특징을 가짐  

DP 적용 접근 방법
1. 최적해 구조의 특성 파악 => 부분문제로 나누기
2. 최적해의 값을 재귀적으로 정의 => 점화식 도출
3. 상향식 방법으로 최적해 계산 => 가장 작은 문제부터 해를 구한 뒤 테이블에 저장

ex) 피보나치 dp 적용
```java
public static int fibo(int n) {
  dp[0] = 0;
  dp[1] = 1;

  for(int i = 2;i<=n;i++) {
    dp[i] = dp[i-1] + dp[i-2];
  }

  return dp[n];
}
```
