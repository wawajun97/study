# DP

DP(Dynamic Programming)는 작은 부분 문제의 답을 이용해 더 큰 문제를 해결하는 알고리즘 기법이다.

## 기본 아이디어

1. 재귀 호출은 하향식(top-down) 접근이다.
2. 메모이제이션은 이전 계산 결과를 저장해 재사용하는 방법이다.

## 예시: 피보나치 수열

### 1. 순수 재귀

- 같은 계산이 여러 번 반복된다.

```java
public static int fibo(int n) {
  if(n < 2) return 1;

  return fibo(n-1) + fibo(n-2);
}
```

### 2. 메모이제이션

- 이미 계산한 값을 저장하므로 중복 계산을 줄일 수 있다.
- 대신 `memo` 배열 같은 추가 메모리가 필요하다.
- 재귀를 사용하므로 입력이 크면 스택 오버플로우 가능성이 있다.

```java
public static int fibo(int n) {
	if(memo[n] != -1) return memo[n];

	memo[n] = fibo(n-1) + fibo(n-2);

	return memo[n];
}
```

## DP란 무엇인가

DP는 작은 부분 문제의 답을 먼저 구해 저장해 두고, 이를 이용해 더 큰 문제를 해결하는 방식이다.

## DP가 적용되는 조건

1. 중복 부분 문제 구조가 있어야 한다.
2. 최적 부분 구조 또는 경우의 수 구조가 있어야 한다.

즉, 어떤 문제의 최적해를 구성하는 작은 문제들의 해도 같은 성질을 가져야 한다.

## DP 적용 절차

1. 문제의 최적 구조를 파악한다.
2. 부분 문제로 나눈다.
3. 점화식을 만든다.
4. 가장 작은 문제부터 값을 채운다.

## 예시: 피보나치 DP

```java
public static int fibo(int n) {
  dp[0] = 0;
  dp[1] = 1;

  for(int i = 2;i<=n;i++) {
    dp[i] = dp[i-1] + dp[i-2];
  }

  return dp[n];
}
```

## 예시: 계수 값 구하기

`(x+y)^n`을 전개했을 때 `x^k y^(n-k)`의 계수는 `nCk`이다.

## 조합

```java
//재귀
public static int comb(int n, int k) {
	if(n==k || n==0) return 1;

	return comb(n-1,k-1) + comb(n-1,k);
}

//dp
public static int comb(int n, int k) {
	for(int i =0;i<=n;i++) {
		int end = Math.min(i,k);
		for(int j = 0;j<=end;j++) {
			if(j == 0 || j == i) b[i][j] = 1;
			else b[i][j] = b[i-1][j-1] + b[i-1][j];
		}
	}

	return b[n][k];
}

//얘는 그냥 조합 구하는 방법
public static long comb(int n, int r) {
	long result = 1;

	for (int i = 1; i <= r; i++) {
		result = result * (n - r + i) / i;
	}

	return result;
}
```

## DP 풀이 순서

1. 문제를 정확히 이해하고, 최적해 문제인지 경우의 수 문제인지 구분한다.
2. 부분 문제를 정의한다.
3. 상태를 어떻게 표현할지 정한다.
   - 예: `d[i]`, `d[i][j]`
4. 상태 간 관계를 분석해 점화식을 만든다.
5. 테이블을 만들고 base case를 초기화한 뒤 값을 채운다.
