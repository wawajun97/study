리스트 : 순서를 가진 데이터의 집합을 가리키는 추상자료형

1. 순차리스트 : 배열을 기반으로 구현된 리스트 -> 검색에 강점, 삽입/삭제에 약점
2. 연결리스트 : 메모리의 동적할당을 기반으로 구현된 리스트 -> 삽입/삭제에 강점, 검색에 약점

Linked List : 자료의 논리적인 순서와메모리 상의 물리적인 순서가 일치하지 않고 개별적으로 위치하고 있는 각 원소를 연결하여 배열을 생성
-> 자료구조의 크기를 동적으로 조정할 수 있어 메모리의 효율적인 사용 가능

노드 : 하나의 원소
 - 데이터 필드 : 원소의 값 저장
 - 링크 필드 : 다음 노드의 참조값 저장
헤드 : 연결 리스트의 첫 노드에 대한 참조값

단순 연결 리스트 : 노드를 한 방향으로 연결(링크 한개)
이중 연결 리스트 : 노드의 연결을 양방향으로 연결(링크 두개)
원형 연결 리스트 : 첫번째 노드와 마지막 노드를 연결

연결 구조
- 노드가 하나의 링크 필드에 의해 다음 노드와 연결된 구조
- 헤드가 가장 앞의 노드를 가리킴
- 링크 필드가 Null인 노드가 연결 리스트의 가장 마지막 노드

첫번째에 노드 삽입
1. 노드 객체 생성 후 데이터 입력
2. 헤드에 값을 생성한 노드의 링크에 입력
3. 헤드의 값을 생성한 노드의 링크로 변경

마지막에 노드 삽입
1. 노드 객체 생성 후 데이터 입력
2. 헤드를 시작으로 마지막 노드 찾기
3. 원래 마지막 노드였던 링크에 생성한 노드의 참조값 입력

중간에 노드 삽입
1. 노드 객체 생성 후 데이터 입력
2. 넣고 싶은 노드의 선행노드 찾기
3. 생성한 노드의 링크를 선행노드의 링크로 변경
4. 선행노드의 링크를 생성한 노드의 참조값으로 변경

중간에 노드 삭제
1. 선행노드의 링크를 삭제할 노드의 링크로 변경

첫번째 노드 삭제
1. 삭제할 노드의 링크 필드를 리스트의 헤드에 복사

마지막 노드 삭제
1. 마지막 노드의 선행노드의 링크를 Null로 변경

LinkedList를 활용하여 스택 구현
- 계속 첫번째에 데이터를 삽입

슬라이딩 윈도우 : 연속된 값을 계산할 때 사용 -> 한칸씩 밀면서 계산
- 각 윈도우의 공통된 교집합의 정보를 재사용하고 차이가 나는 양쪽 끝 원소만 갱신
- 배열이나 리스트의 요소의 일정 범위의 값을 비교할 때 사용하면 매우 유용
