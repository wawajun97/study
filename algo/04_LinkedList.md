# 리스트

리스트는 순서를 가진 데이터의 집합을 표현하는 추상 자료형이다.

1. 순차 리스트: 배열 기반으로 구현되며, 검색에 강하고 삽입/삭제에는 약하다.
2. 연결 리스트: 동적 메모리 할당 기반으로 구현되며, 삽입/삭제에 강하고 검색에는 약하다.

## Linked List

연결 리스트는 자료의 논리적 순서와 메모리상의 물리적 순서가 일치하지 않아도 되는 구조이다.
각 원소가 개별적으로 존재하고, 링크를 통해 다음 원소와 연결된다.

- 크기를 동적으로 조절할 수 있어 메모리를 비교적 효율적으로 사용할 수 있다.

## 기본 구성

- 노드(node): 하나의 원소
  - 데이터 필드: 원소의 값을 저장
  - 링크 필드: 다음 노드의 참조를 저장
- 헤드(head): 연결 리스트의 첫 노드를 가리키는 참조값

## 연결 리스트 종류

- 단순 연결 리스트: 노드를 한 방향으로 연결한다.
- 이중 연결 리스트: 노드를 양방향으로 연결한다.
- 원형 연결 리스트: 마지막 노드가 첫 번째 노드와 연결된다.

## 연결 구조

- 각 노드는 링크 필드를 통해 다음 노드와 연결된다.
- 헤드는 가장 앞에 있는 노드를 가리킨다.
- 링크 필드가 `null`인 노드가 연결 리스트의 마지막 노드이다.

## 첫 번째에 노드 삽입

1. 새 노드 객체를 만들고 데이터를 저장한다.
2. 새 노드의 링크를 기존 헤드로 연결한다.
3. 헤드를 새 노드로 변경한다.

## 마지막에 노드 삽입

1. 새 노드 객체를 만들고 데이터를 저장한다.
2. 헤드부터 시작해 마지막 노드를 찾는다.
3. 기존 마지막 노드의 링크를 새 노드로 연결한다.

## 중간에 노드 삽입

1. 새 노드 객체를 만들고 데이터를 저장한다.
2. 삽입 위치 바로 앞의 선행 노드를 찾는다.
3. 새 노드의 링크를 선행 노드의 기존 링크로 연결한다.
4. 선행 노드의 링크를 새 노드로 변경한다.

## 중간 노드 삭제

1. 선행 노드의 링크를 삭제할 노드의 다음 노드로 변경한다.

## 첫 번째 노드 삭제

1. 헤드를 삭제할 노드의 다음 노드로 변경한다.

## 마지막 노드 삭제

1. 마지막 노드의 선행 노드를 찾는다.
2. 선행 노드의 링크를 `null`로 변경한다.

## LinkedList로 스택 구현

- 가장 앞에 데이터를 계속 삽입하면 스택처럼 사용할 수 있다.

## 슬라이딩 윈도우

슬라이딩 윈도우는 연속된 구간의 값을 계산할 때 사용하는 기법이다.

- 한 칸씩 이동하면서 계산한다.
- 공통으로 겹치는 구간의 정보는 재사용하고, 양쪽 끝 원소만 갱신한다.
- 배열이나 리스트에서 일정 구간을 반복해서 비교할 때 효율적이다.
- 완전탐색으로는 `O(nm)`이 걸릴 수 있지만, 슬라이딩 윈도우를 사용하면 `O(n)`까지 줄일 수 있다.

## 이중 연결 리스트

이중 연결 리스트는 양쪽 방향으로 순회할 수 있도록 `prev`, `next` 링크를 모두 가진다.

- 첫 노드의 `prev`는 `null`이다.
- 마지막 노드의 `next`는 `null`이다.
- 노드 삭제 시 이전 노드를 다시 탐색하지 않아도 된다.

## 이중 연결 리스트 삽입

1. 새 노드 객체를 만들고 데이터를 저장한다.
2. 새 노드의 `next`를 이전 노드의 `next`로 연결한다.
3. 새 노드의 `prev`를 다음 노드의 `prev`로 연결한다.
4. 이전 노드의 `next`를 새 노드로 변경한다.
5. 다음 노드의 `prev`를 새 노드로 변경한다.

## 이중 연결 리스트 삭제

1. 이전 노드의 `next`를 삭제할 노드의 `next`로 변경한다.
2. 다음 노드의 `prev`를 삭제할 노드의 `prev`로 변경한다.
3. 삭제할 노드의 링크를 `null`로 처리한다.
