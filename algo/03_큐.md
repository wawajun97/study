큐
- 스택과 마찬가지로 삽입과 삭제의 위치가 제한적인 자료구조 : FIFO(First In First Out)
- ArrayDeque 클래스를 주로 많이 사용한다 -> 삽입, 삭제가 정해져있기 때문
  
기본연산
enQueue : 삽입
deQueue : 삭제

front와 rear로 큐를 관리
front : 맨 앞 원소 인덱스
rear : 맨 뒤 원소 인덱스
=> 큐가 비어있으면 front = rear = -1

삽입 시 rear + 1, 삭제 시 front + 1

주요 메소드
- offer() : 큐에 삽입(또는 add())
- poll() : front의 값 확인하고 front값 삭제
- peek() : front의 값 확인
- isEmpty() : 큐가 비어있는지 확인
- size() : 큐 사이즈

Deque : Double ended Queue -> 앞 뒤로 모두 삽입/삭제 가능

버퍼 : 데이터를 한 곳에서 다른 한 곳으로 전송하는 동안 일시적으로 그 데이터를 보관하는 메모리의 영역
- FIFO 방식의 큐가 사용됨
