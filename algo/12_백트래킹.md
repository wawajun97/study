# 백트래킹

백트래킹은 완전탐색을 수행하되, 가능성이 없는 경우를 미리 잘라내어 탐색 횟수를 줄이는 방법이다.

- 끝까지 가더라도 해답이 될 수 없는 경우는 더 이상 탐색하지 않는다.
- 어떤 노드가 유망하지 않다고 판단되면 부모 노드로 되돌아가 다른 경로를 탐색한다.
- 유망하다는 것은 해답이 될 가능성이 있다는 뜻이다.
- 가지치기(pruning)는 유망하지 않은 경로를 더 이상 탐색하지 않는 것이다.

## 백트래킹 절차

1. 상태 공간 트리를 DFS 방식으로 탐색한다.
2. 각 노드가 유망한지 검사한다.
3. 유망하지 않으면 부모 노드로 돌아가 다른 경로를 탐색한다.

## 백트래킹 vs DFS

- DFS는 가능한 경로를 끝까지 모두 탐색한다.
- 백트래킹은 DFS를 기반으로 하되, 가지치기를 통해 불필요한 탐색을 줄인다.

## 예시: N-Queen

N-Queen은 서로 공격하지 않도록 `N`개의 퀸을 배치하는 문제이다.

- 같은 열인지 확인하는 배열: 크기 `N`의 `boolean` 배열을 사용하고, 열 인덱스를 기준으로 체크한다.
- 왼쪽 대각선(`/`) 확인 배열: 크기 `N * 2`의 배열을 만들고 `row + col`을 인덱스로 사용한다.
- 오른쪽 대각선(`\`) 확인 배열: 크기 `N * 2`의 배열을 만들고 `N + (row - col)`을 인덱스로 사용한다.
- 위 세 배열이 모두 `false`이면 현재 위치에 퀸을 놓을 수 있다.

```java
public class NQueenTest {
	public static BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
	public static int N,totalCnt;
	public static boolean[] col, slash,bSlash;
	public static void main(String[] args) throws NumberFormatException, IOException {
		N = Integer.parseInt(br.readLine());

		totalCnt = 0;

		col = new boolean[N+1]; // 열
		slash = new boolean[N*2 + 1]; // 왼쪽 대각선 /
		bSlash = new boolean[N*2 + 1]; // 오른쪽 대각선 \

		setQueen(1);

		System.out.println(totalCnt);
	}

	public static void setQueen(int row) {
		if(row == N+1) {
			totalCnt++;
			return;
		}

		for(int c =1;c<=N;c++) {
			// 열, 대각선을 확인해서 놓치 못하면 더 이상 탐색 x -> 가지치기
			if(col[c] || slash[c+row] || bSlash[N-(row-c)]) continue;

			col[c] = slash[row+c] = bSlash[N-(row-c)] = true;
			setQueen(row+1);
			col[c] = slash[c+row] = bSlash[N-(row-c)] = false;
		}
	}

}
```
